# 应季食谱参考网页设计稿

日期：2026-07-20

## 1. 项目目标

将《成人肥胖食养指南（2024年版）》中的地区食谱示例整理为一个适合日常查阅的响应式网页。网页以“当前季节”为默认入口，让用户快速找到适合当下季节的全天食谱，并通过地域和能量档进行组合筛选。

网页是原文内容的结构化参考工具，不替代指南原文、医生诊断或个体化营养方案。

## 2. 已确认的产品决策

- 默认入口为当前季节。
- 用户可以手动切换春、夏、秋、冬四季。
- 地域和能量档作为后续筛选条件。
- 采用“列表快速参考 + 点击查看详情”的两层结构。
- 第一版只整理用户提供的指南原文，不额外编造新食谱。
- 网页必须适配手机、平板和桌面端。
- 页面显著位置展示健康参考与专业咨询提示。

## 3. 信息架构

### 3.1 首页 / 食谱库

首页从上到下包含：

1. 品牌标题与页面说明。
2. 当前季节标题，例如“夏季食谱参考”，并显示当前季节的短说明。
3. 季节切换控件：春、夏、秋、冬；按月份自动高亮当前季节，同时允许手动切换。
4. 组合筛选：地域、能量档（约 1200 / 1400 / 1600 kcal）。
5. 食谱卡片列表。
6. 健康提示、内容范围说明和原文出处。

### 3.2 食谱卡片

每张卡片提供快速决策所需的信息：

- 所属地域和季节。
- 全天食谱的能量档。
- 早餐、午餐、加餐、晚餐名称缩略预览。
- 总能量及主要营养数据。
- “查看详情”操作。

### 3.3 食谱详情

详情视图包含：

- 食谱标题、地域、季节、能量档。
- 总能量、蛋白质、碳水化合物、脂肪及供能比例。
- 早餐、午餐、加餐、晚餐分区。
- 每道食物的食材、份量和原文注释。
- 食药物质标记及相关说明。
- 原文页码或来源标识。
- 健康参考提示。

桌面端使用右侧详情面板或独立详情视图；移动端使用底部抽屉，关闭后保留原筛选状态。

## 4. 数据模型

数据按“季节 → 地域 → 能量档 → 全天食谱”组织。建议的单条记录结构如下：

```text
recipePlan
  id
  season
  region
  energyKcal
  meals
    breakfast
    lunch
    snack
    dinner
  nutrition
    protein
    carbohydrate
    fat
    ratios
  notes
  sourcePage
```

数据范围以附录 3 的不同地区四季食谱示例为主，预计覆盖 7 个地区、4 个季节和 3 档能量示例。所有食材份量与营养数据以 PDF 原文为准。

## 5. 视觉方向

采用“自然食养 + 轻量工具感”的视觉语言：

- 米白色作为背景。
- 深松绿色作为主要文字、导航和操作色。
- 橙红色强调当前季节、主要按钮和重要状态。
- 少量金黄色用于营养数据和辅助提示。
- 标题使用有温度的中文展示字体，正文使用清晰的系统字体。
- 不依赖外部图片；使用季节色块、食材小图形和简洁装饰，保持轻量与内容一致性。
- 卡片使用柔和边框、圆角和清晰的信息分组，避免过度装饰影响查阅速度。

## 6. 响应式与交互规则

- 手机端：单列卡片；筛选控件横向滚动；详情使用底部抽屉；触控区域足够大。
- 平板端：双列卡片；详情使用侧边面板。
- 桌面端：三列卡片；详情使用右侧固定面板或独立视图。
- 当前季节根据月份自动确定：3–5 月春季、6–8 月夏季、9–11 月秋季、12–2 月冬季。
- 手动切换季节后，当前筛选状态保持可见。
- 筛选条件写入 URL，便于分享和恢复浏览状态。
- 无匹配结果时展示清晰空状态，并提供一键清除筛选。
- 支持键盘操作、焦点状态和足够的颜色对比度。

## 7. 内容安全与质量规则

- 不新增指南中没有的食谱、治疗功效或诊断结论。
- 克、毫升、颗等单位统一排版，但不改变原文份量。
- PDF 转文本可能存在数字、单位或标点异常；入库前必须对照 PDF 页面核对，无法确认时以 PDF 页面为准并保留原始注释。
- 页面提示内容仅用于成人肥胖人群的一般食养参考。
- 孕妇、乳母、老年人、慢性病或并发症人群需在医生或营养专业人员指导下使用。
- 食药物质单独标识，避免被误解为药物处方。

## 8. 验收标准

### 内容

- 食谱名称、食材份量、能量和营养数据可追溯到 PDF 原文。
- 四季、地域和三档能量筛选结果准确。
- 详情视图完整展示四餐和原文注释。

### 交互

- 默认季节与系统日期一致。
- 可手动切换季节并组合地域、能量档筛选。
- 打开和关闭详情不会丢失筛选状态。
- URL 可以恢复筛选结果。
- 无结果时可清除筛选并恢复列表。

### 响应式

- 手机、平板、桌面三种布局均无横向溢出。
- 卡片、筛选控件和详情面板在触控设备上可操作。
- 关键文字和按钮在小屏上不被截断。

### 可访问性与质量

- 键盘可以完成季节切换、筛选、打开和关闭详情。
- 焦点状态清晰，颜色对比度可读。
- 主要内容具备清晰的标题层级和语义标签。

## 9. 取舍与非目标

- 第一版不做账号、收藏、打卡、个性化计算或后端服务。
- 第一版不把网页扩展成医疗问诊或诊断工具。
- 第一版优先完成内容准确、筛选顺手和响应式阅读，再考虑图片、收藏和个性化功能。
