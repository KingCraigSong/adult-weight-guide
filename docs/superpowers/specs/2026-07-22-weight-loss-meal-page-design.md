# 减脂餐页面设计稿

日期：2026-07-22

## 1. 目标

在现有“应季食谱参考”Vue 页面中新增一个独立的“减脂餐”内容入口，整理公开网页菜谱为可查阅的轻量工具。该页面用于菜谱灵感和一般饮食参考，不提供个体化减重方案，也不替代医生或营养专业人员建议。

## 2. 入口与交互

- 顶部品牌区增加“应季食谱 / 减脂餐”两个入口。
- 使用 URL 查询参数 `page=weight-loss` 表示减脂餐页面，默认仍打开应季食谱页面。
- 减脂餐页面支持目标热量（全部、≤300 kcal、301–450 kcal、>450 kcal）、餐次（早餐、午餐、晚餐、加餐）、烹饪方式（蒸煮、煎炒、汤羹、拌饭）、菜谱类型（沙拉、杂粮饭盒、冷面、全麦主食等）和准备方式组合筛选。
- 增加“只看便当友好”选项，重点覆盖前一晚完成、前一晚备料、早上快做和无需加热的单位午餐场景。
- 筛选状态同步到 URL，详情打开和关闭不丢失筛选状态。
- 移动端继续使用底部详情抽屉，桌面端使用右侧抽屉。

## 3. 内容范围

当前录入 33 道公开菜谱，覆盖鸡胸、虾仁、鱼类、牛肉、鸡蛋、豆腐等蛋白质，以及沙拉、藜麦、糙米、荞麦面、全麦面包、红薯等便当主食组合。

每道菜包含：菜名、餐次、烹饪方式、菜谱类型、准备方式、估算热量、食材、步骤概述、标签、携带方式、保存建议、来源标题和来源 URL。热量优先使用来源页面明确标注的数值；没有明确标注时使用保守估算并标记“估算”。

来源以公开菜谱页为主，包括下厨房、香哈和健康教育材料。页面只展示简短改写后的食材与步骤，提供原文链接供核对，不复制来源的完整文字。

## 4. 视觉与可访问性

- 复用现有米白、松绿色、橙红色视觉 token，减脂餐页使用更明亮的薄荷绿辅助色区分入口。
- 使用语义标题、筛选标签、可键盘操作按钮和清晰的焦点状态。
- 无匹配结果时提供清除筛选操作。
- 健康提示明确说明估算热量不等于个体实际能量需求，并提示过敏、慢性病及特殊人群咨询专业人员。

## 5. 验收标准

- 可在两个页面入口间切换，刷新或复制 URL 后仍能恢复页面和筛选条件。
- 33 道减脂餐可按热量、餐次、烹饪方式、菜谱类型和准备方式筛选并打开详情。
- 详情抽屉可关闭，关闭后筛选条件保留。
- 桌面、移动端无横向溢出，按钮和筛选控件可操作。
- 单元测试覆盖筛选和 URL 状态；生产构建通过。
